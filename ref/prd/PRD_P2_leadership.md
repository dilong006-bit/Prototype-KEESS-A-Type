## I. 목적 · 전략
- P2는 KEESS 4대 필러 중 **리더십·조직** 필러. 핵심 메시지 = "**역할이 바뀌는 순간마다, 다른 리더십이 필요하다**" — 단발성 교육이 아니라 성장단계별 리더십을 **진단→역량개발→현업적용→조직확산→조직문화 정착**의 하나의 체계로 설계.
- A안 정체성 = **Journey(성장 여정) 중심**. 6단계 성장 로드맵을 페이지의 주력 서사로 세우고, 6대 역량·리더십 휠·성장시점 스텝퍼·조직진단으로 근거를 보강.
- 목표: 체계·역량·진단 근거를 제시해 **리더십 도입 문의 리드**를 인입.
- 범위 외: 정부지원 환급·과정리스트 다운로드(B안 전용). 전사 인증·수상·고객사(홈 전용). P2엔 없음.

## III. 인터랙션 (시안 동작 보존, 홈 모션 규칙으로 통일)
- **Nav solid + SubNav 스크롤스파이**: 스크롤 시 nav 배경 전환, SubNav 등장(고민·성장여정·6대 역량·Framework·오프라인·조직진단·차별점·도입문의) + 현재 섹션 활성.
- **Reveal**: `.r` 요소 뷰포트 진입 시 등장(홈 `useReveal`, IntersectionObserver th **.16** 통일 — 시안 .12는 정규화). reduced-motion 대응.
- **리더십 휠(framework)**: 섹션 뷰포트 진입 시 6세그먼트 순차 등장(원형 SVG). 세그먼트 hover 시 살짝 바깥 이동, **클릭 시** 상세 패널(hint·name·desc·세부역량 칩) 갱신 + 중앙 라벨 갱신. 키보드 접근 대응.
- **성장시점 스텝퍼(offline)**: 4스테이지 버튼 클릭 → 하위 프로그램 카드(세부역량 칩) 패널 렌더. 초기값 STAGE 1. `aria-selected`.
- **모달 2종**:
  - **도입 문의(공통 Modal)**: nav·hero CTA → 안내 후 "문의 폼으로 이동"(#inq 스크롤). ESC·포커스 트랩.
  - (P2 시안엔 상담/가이드 이원 모달 없음 — 홈/P1과 일관되게 도입 문의는 #inq 폼으로 수렴. 필요 시 공통 ConsultModal 재사용)
- **문의 폼**: 회사·담당자·연락처 필수 검증 → 성공 상태("문의가 접수되었습니다"). 백엔드 없음.

## IV. 비기능 요구
- **틴트**: 페이지 wrapper `.tint-p2`(bg#FDFBFC surface#FCEBF2 ink#181318 line#F0E3EA), `--maxw:1180px`. 필러색(#E91E63)은 포인트에만(eyebrow·강조·CTA·휠/타임라인 액센트).
- **색 스케일(데이터 시각화 예외)**: 리더십 휠 6세그먼트·성장 타임라인 레일은 **--p2 기반 마젠타 계열 스케일**을 데이터 시각화 포인트로 허용(페이지 톤 오염 아님). 그 외 카드·배경·본문은 중립 베이스+틴트 유지. `오프라인` 배지는 홈 배지 토큰 사용(시안의 보라 #6A3FB8 신설 금지 — Badge tone으로 표현).
- **폰트**: 전 페이지 통일 Pretendard Variable(self-host) 상속. 신규 서체 금지.
- **공통 재사용**: Nav(current='leadership')·Footer·SubNav·Modal·Button·Card·Chip·Badge·Reveal·Section·SectionHeader 기존 컴포넌트.
- **모션**: 홈 useReveal(th .16)·hover(translateY -2~3px + shadow) 규칙, 등장 .7~.9s, reduced-motion 대응.
- **반응형**: 확정 브레이크포인트만(920/900/880/820/760/720/620/520/480). 6스텝 타임라인→2열→1열, 휠 축소, 스텝퍼 세로 전환. 3디바이스 정상.
- **접근성**: 모달 포커스 트랩·ESC, 휠 세그먼트·스텝퍼 키보드 조작·`aria-selected`, 폼 라벨·인라인 에러, SVG `role/aria-label`.
