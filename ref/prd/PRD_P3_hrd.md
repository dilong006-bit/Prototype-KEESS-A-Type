## I. 목적 · 전략
- P3은 KEESS 4대 필러 중 **HRD 통합 솔루션** 필러. 핵심 메시지 = "**여러 곳에 맡기던 HRD를, 한 곳에서**" — 시스템·운영·제작을 벤더 확장 없이 KG 단일 창구로 통합 운영.
- A안 정체성 = **통합 서사형**. 담당자 고충(따로따로 관리)에서 출발해 3축 풀스택(시스템·운영·제작) → 도입사례 → 차세대 시스템(KGESA) 데모 → 도입절차로 안심을 쌓아 상담 전환.
- 목표: 3축 근거·운영 실적·KGESA 체험을 제시해 **HRD 도입 문의 리드**를 인입.
- 범위 외: 정부지원 환급(B안 전용). 전사 인증·수상은 홈 전용. P3(A안)엔 없음.

## III. 인터랙션 (시안 동작 보존, 홈 모션 규칙으로 통일)
- **Nav solid + SubNav 스크롤스파이**: items(고민·3축 솔루션·시스템·운영·제작·도입사례·차세대·도입절차·도입문의) + 현재 섹션 활성.
- **Reveal**: `.r` 뷰포트 진입 시 등장(홈 `useReveal`, IntersectionObserver th **.16** 통일 — 시안 .12 정규화). reduced-motion 대응.
- **KGESA 데모(#kgesa · 핵심)**:
  - 컨트롤: 테마 2버튼(GNB형 상단메뉴 / LNB형 좌측메뉴), 위젯 리스트 6종(메인 배너·학습 대시보드·과정 리스트·추천 강의·디베이트·플로팅 메뉴) 각 on/off 스위치 + 순서 ↑/↓.
  - 미리보기(FO): 컨트롤 상태에 따라 학습자 화면을 **실시간 재렌더**(테마 레이아웃 전환, 켜진 위젯만 순서대로 표시, 플로팅 버튼 토글). 블록 등장 시 소폭 페이드.
  - 상태 관리 `useState`(테마·위젯 배열). 키보드 조작·`aria-pressed`/`aria-label`. reduced-motion 시 페이드 생략.
- **도입 문의 모달**: 공통 `Modal` → 안내 후 "문의 폼으로 이동"(#inq). nav·hero CTA 트리거.
- **문의 폼(#inq)**: 공통 `InquiryForm` 재사용, 회사·담당자·연락처 필수 검증 → 성공 상태. 백엔드 없음.

## IV. 비기능 요구
- **틴트**: 페이지 wrapper `.tint-p3`(surface#F4EAFA line#ECE0F4 ink#1A1420), `--maxw:1180px`. 필러색(#8B27A8)은 포인트에만(eyebrow·강조·CTA·타임라인/KGESA 액센트).
- **색 스케일(데이터 시각화 예외)**: KGESA FO 배너·프로세스 스텝 넘버·타임라인 노드 등은 **--p3 기반 바이올렛 스케일**을 데이터 시각화/포인트로만 허용. 그 외 카드·배경·본문은 중립 베이스+틴트 유지. 시안의 보라 #8B6FE0·#6A3FB8 등 신설 금지(Badge tone 사용).
- **폰트**: 전 페이지 통일 Pretendard Variable(self-host) 상속. 신규 서체 금지.
- **공통 재사용**: Nav(current='hrd')·Footer·SubNav·Modal·Button·Card·Chip·Badge·Reveal·Section·SectionHeader·InquiryForm.
- **모션**: 홈 useReveal(th .16)·hover(translateY -2~4px + shadow), reduced-motion 대응.
- **반응형**: 확정 브레이크포인트만(880/860/820/760/720/640/520/480). 3축 flex→세로, 목업 split→1열, HOW 타임라인 가로 스크롤, KGESA 데모 2열→1열. 3디바이스 정상.
- **접근성**: 모달 포커스 트랩·ESC, KGESA 컨트롤(테마·스위치·순서) 키보드·ARIA, 폼 라벨·인라인 에러.

## V. 성공 기준 (DoD)
- **구성·인터랙션 = 시안과 일치**(10섹션·KGESA 데모·SubNav), **비주얼 = 홈 디자인 시스템(Design.md)과 일관**(홈·P1·P2와 나란히 대조). → 시안과 픽셀 1:1 일치는 목표 아님.
- KGESA 테마/위젯 on-off/순서 → FO 실시간 반영, 스크롤스파이, 문의 폼 검증/성공, 모달 동작.
- **신규 색토큰·서체·버튼변형 0**(KGESA/타임라인 --p3 파생 스케일만 예외), `.tint-p3` 적용, `next build` 무오류(홈·P1·P2 회귀 없음).
- 공통 Footer(부정훈련 예방/신고) 정상, 3디바이스 정상, 키보드·reduced-motion 대응.
