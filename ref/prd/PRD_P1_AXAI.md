## I. 목적 · 전략
- P1은 KEESS 4대 필러 중 **대표 필러(AX·AI 전환)**. 핵심 메시지 = "**교육이 아니라 전환을 설계한다**"(진단→설계→학습→실행→성과 End-to-End).
- 목표: 진단·프레임워크·성과 근거를 제시해 **AX 진단 상담 / 도입 가이드 리드**를 인입.
- **D안 확정**: 구성(IA·섹션 순서·카피)은 변경 금지. **비주얼은 홈 디자인 시스템(Design.md)으로 재현** — 시안 스타일 이식이 아니라 홈 토큰·컴포넌트로 표현.
- 범위 외: 정부지원 환급·과정리스트 다운로드(이는 B안 전용). P1엔 없음.

## III. 인터랙션 (원본 동작 보존)
- **목표별 경로(scenario)**: 목표 선택 → 우측 패널(추천 경로·칩·CTA) 전환.
- **Framework 단계**: Lv1~5 계단 선택 → 설명(fwDesc)·역량영역(fwAreas) 갱신. "이런 고민이라면 이 단계부터" 문구 매핑.
- **직무 맵**: 포지셔닝 맵 + 직무칩 선택 → 5단계 역량 스텝 갱신.
- **성과 GAP**: 뷰포트 진입 시 pre/post 바 폭 애니메이션(사전<사후).
- **카운트업**: 스탯 숫자 뷰포트 진입 시 카운트업(reduced-motion 시 즉시 최종값).
- **모달 2종**:
  - **가이드 받기(GuideModal)**: 회사·담당자·이메일·개인정보 동의 → 검증 → 성공 상태("가이드를 보내드렸습니다"). 백엔드 없음.
  - **상담 신청(ConsultModal)**: 부문(B2B/B2G)·규모·회사·담당자·이메일·관심영역·연락처·문의 → 검증 → 성공("영업일 1일 내 회신"). 백엔드 없음.

## IV. 비기능 요구
- **틴트**: 페이지 wrapper `.tint-p1`(bg#FCFCFE surface#F4F2FB ink#15131D line#EAE8F2), `--maxw:1180px`. 필러색(#2E1A6B)은 포인트에만.
- **폰트**: 전 페이지 통일 Pretendard Variable(self-host) 상속. 신규 서체 금지.
- **공통 재사용**: Nav(current='ax-ai')·Footer·Modal·Button·Reveal·MetricStat·Section 등 기존 컴포넌트.
- **모션**: 기존 useReveal(th .16)·hover 규칙, reduced-motion 대응.
- **반응형**: 확정 브레이크포인트만. 3디바이스 정상.
- **접근성**: 모달 포커스 트랩·ESC, 선택 UI 키보드, 폼 라벨·검증.
